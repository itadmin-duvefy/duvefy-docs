---
title: "Cloning an Agent"
description: "Create an exact copy of an agent to use as a starting point for a new one."
---

## Overview

Cloning creates an independent copy of an existing agent — including all configuration, topics, and footer links. The clone is completely separate from the original; editing one does not affect the other.

---

## How to clone

<Steps>
  <Step title="Go to the Agents list">
    In the left sidebar, click **Agents**.
  </Step>
  <Step title="Find the agent to clone">
    Locate the agent card you want to copy.
  </Step>
  <Step title="Click the clone icon">
    Click the **copy icon** on the agent card (visible to owners and admins only). A confirmation modal appears.
  </Step>
  <Step title="Confirm">
    Click **Clone Agent** in the modal. The new agent appears in your list immediately.
  </Step>
</Steps>

---

## What gets copied

| Item | Copied? |
|------|:-------:|
| Agent name (with "Copy of" prefix) | Yes |
| Mission statement | Yes |
| Agent type | Yes |
| Knowledge base (text, URLs, PDFs) | Yes |
| Tone & rules | Yes |
| Widget styling | Yes |
| Topics | Yes |
| Topic actions | Yes |
| Footer links | Yes |
| Active/inactive state | No — clone starts **inactive** |
| Conversation history | No |
| Analytics data | No |

---

## Use cases

- **A/B testing** — Clone an agent and modify the mission or tone, then compare engagement across both
- **Seasonal campaigns** — Clone your standard agent and adjust it for a specific event or campaign period
- **New markets** — Clone a working agent and adapt the knowledge base for a different audience

---

## After cloning

The cloned agent starts **inactive**. Before going live:
1. Rename it to reflect its new purpose
2. Update the knowledge base if needed
3. Review topics and actions
4. Set up a new embed script for the new agent (each agent has its own embed URL)
5. Toggle it **Active** when ready
