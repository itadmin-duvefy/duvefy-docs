## What counts as a message?

Every message sent by a visitor to an agent counts toward the monthly message limit. Agent responses do not count separately.

The current month's usage is visible on the **Dashboard** as the "Messages This Month" card, including a progress bar.

<Warning>
  When your organization reaches **80% of its message limit**, a prompt appears on the dashboard encouraging you to upgrade. At 100%, the agent stops responding until the next billing period or the limit is increased.
</Warning>

## Overage behavior

| Behavior | What happens |
|----------|-------------|
| **Blocked** | Agent stops responding when the monthly limit is reached. Visitors see a polite "unable to respond" message. |
| **Billed** | Messages beyond the limit are metered and charged at an agreed per-message rate. |
| **Negotiated** | Enterprise customers negotiate custom overage terms with Duvefy. |

## Custom limits

Duvefy's operations team can apply custom message and agent limits to individual organizations, overriding the plan defaults. This is useful for organizations that need more capacity without changing their base plan.

## Plan features explained

### CRM integrations (Growth+)
Connect Salesforce, HubSpot, or Zoho to allow topics to create, update, and query CRM records.

### Topic Actions (Scale+)
Unlock configurable actions inside topics: Salesforce Flow, Apex Class, HTTP requests, and Send Email. Actions are the building blocks for sophisticated automated workflows.

### Bring Your Own Key — BYOK (Scale+)
Use your own LLM API key (OpenAI, Anthropic, or Google) instead of Duvefy's default. Useful for cost control or model preference. See [LLM Settings](/settings/llm).

### Audit Logs (Scale+)
A forensic trail of sensitive admin operations (team changes, CRM connections, key rotations, etc.) accessible to org owners at **Settings → Audit Logs**. See [Audit Logs](/security/audit-logs).
